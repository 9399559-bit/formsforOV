# Эпик: Сбор лидов на выставке (iZBURG) → Bitrix24 — v3

> Источник истины: `brief-vystavka-lidy-izburg.md` (требует синхронизации с этой версией). Это отдельная система iZBURG, **не** платформа ServiceDom — каноничные документы `00–13` здесь не применяются.

**Изменения в v3 (решения по GAP v2):**
- US-5: вход менеджера — по **личной ссылке**, которую админ назначает менеджеру в своём кабинете.
- Связь лида с подписанной бумагой — по **телефону + дате** (в системе и в бумажном архиве); № бланка не нужен.
- Обязательный контакт — **минимум один** (телефон или email).
- Ретеншен, удаление и отзыв согласия — **вручную**, вне системы.

*(v2: убраны реклама и второе согласие, новые поля Bitrix, хранение фото; источник/форма помечаются существующими полями; обязательны только Имя + контакт + ПДн.)*

---

## 1. Название

Сбор лидов на выставке (iZBURG): три канала → Bitrix24, с привязкой к менеджеру и соблюдением 152-ФЗ.

## 2. Цель / бизнес-ценность

Не терять контакты посетителей выставки и быстро доводить их в Bitrix24 (Лиды) — с минимумом ручного ввода и без нарушений 152-ФЗ. Каждый лид закреплён за собравшим менеджером (он же владелец и работает с ним). Администратор видит аналитику по каналам и менеджерам.

Ценность: больше собранных лидов и меньше потерь, меньше ошибок ввода контактов, юридическая чистота согласия, прозрачность вклада каждого менеджера.

## 3. Контекст

- iZBURG — отдельное юрлицо; система самостоятельная.
- Три канала: веб-форма (QR), Telegram (Mini App), бумажный бланк с подписью + вечерний занос менеджером через распознавание по фото.
- Веб-форма — на **существующем сайте iZBURG** (там опубликована политика ПДн). Если админ сайта не даст добро — на сайте дашбордов.
- Все каналы используют один бэкенд и один маппинг полей Bitrix; различаются пометкой формы.
- **Одно согласие — ПДн (обязательное).** Рекламного согласия нет.
- Набор полей — предполагаемый, **на утверждении**. Обязательны только Имя + хотя бы один контакт (телефон или email) + ПДн; остальные опциональны.
- Выставочный источник помечаем существующим полем «Источник трафика» (значение «Выставка»/«Мероприятие»); форму (веб/тг/бумага) — желательно, существующим текстовым полем «Дополнительно об источнике». Новые поля не создаём. *(выбор полей — на подтверждении)*
- Хостинг и распознавание (OCR) — на российской инфраструктуре.

## 4. Объём

**Входит:**
- Веб-форма (браузер по QR) на сайте iZBURG: набор полей + **одна** обязательная галочка ПДн; обязательны только Имя + контакт.
- Telegram-канал: Mini App-форма с проверкой подписи; захват Telegram-контакта.
- Режим менеджера: вход по личной ссылке, пакетная загрузка фото бланков, распознавание, проверка/правка, подтверждение → лид на менеджера.
- Отправка в Bitrix24 (Лиды): маппинг полей, мультиполя телефон/email, запись факта согласия с датой, пометка источника и формы существующими полями.
- Дедупликация по телефону/email.
- Администратор: аналитика + управление аккаунтами менеджеров (включая назначение/отзыв личных ссылок), маппинг на пользователей Bitrix, настройка списков значений.
- Стандартизированный шаблон бумажного бланка (галочки + клеточки под телефон + блок согласия с подписью).

**Не входит:**
- Реклама и согласие на рекламу.
- Хранение фото бланков в системе (фото только для распознавания). Подписанная бумага физически уходит в бухгалтерию.
- Создание новых полей в Bitrix.
- Защита веб-формы от спама.
- **Автоматический ретеншен/удаление данных и обработка отзыва согласия — выполняются вручную, вне системы.**
- Маркировка/регистрация исходящих звонков; уведомление в Роскомнадзор; юр. формулировки согласия (зона юриста iZBURG).
- Обработка лидов после попадания в Bitrix (воронка, обзвон).
- Старший менеджер как отдельная системная роль (в системе он = менеджер).

## 5. Декомпозиция (пользовательские истории)

**Предполагаемый набор полей (на утверждении):** Номер телефона, Наличие участка, Локация, Линейка, Проект, Дата начала строительства, Источник финансирования, Бюджет строительства, Электронная почта, Удобный способ коммуникации, Дополнительные комментарии. Плюс Имя. **Обязательны:** Имя + хотя бы один контакт (телефон или email) + галочка ПДн. Остальные — опциональны.

### Посетитель
- **US-1.** Отсканировать QR и заполнить форму в браузере (сайт iZBURG), без установки приложений. — *Критерии:* QR ведёт на веб-форму; открывается в любом мобильном браузере; обязательны только Имя + контакт + ПДн, остальное можно пропустить; отправка создаёт лид с пометкой «веб».
- **US-2.** Заполнить форму в Telegram. — *Критерии:* ссылка/QR открывает бота → кнопка открывает Mini App; подпись Telegram проверяется на бэкенде; лид создаётся с пометкой «telegram» и захваченным Telegram-контактом.
- **US-3.** Подтвердить согласие на обработку ПДн. — *Критерии:* одна обязательная галочка ПДн — без неё «Отправить» неактивна; текст политики доступен (ссылка на политику сайта iZBURG).
- **US-4.** Получить подтверждение отправки. — *Критерии:* при успехе — подтверждение; при ошибке — понятное сообщение и возможность повторить.

### Менеджер
- **US-5.** Войти в систему по личной ссылке, чтобы лиды закреплялись за мной. — *Критерии:* админ назначает менеджеру уникальную личную ссылку в своём кабинете; открытие ссылки идентифицирует менеджера и даёт доступ к его рабочему месту; ссылка обрабатывается как учётные данные (не пересылать); админ может отозвать/перевыпустить ссылку.
- **US-6.** Загрузить фото нескольких бланков сразу. — *Критерии:* множественная загрузка; бланки встают в очередь на проверку.
- **US-7.** Чтобы система распознала бланк и предзаполнила поля. — *Критерии:* галочки и структура распознаются; контактные поля предзаполняются; распознавание на РФ-инфраструктуре; фото только для распознавания и **не сохраняется**.
- **US-8.** Проверить и поправить распознанное перед отправкой. — *Критерии:* экран проверки показывает все поля; поля редактируемы; акцент на Имя/телефон/email; подтверждение отправляет лид.
- **US-9.** Чтобы подтверждённый лид ушёл в Bitrix на меня, с пометкой «бумага» и выставочного источника. — *Критерии:* `ASSIGNED_BY_ID` = менеджер; форма = «бумага»; источник = «Выставка»; записаны телефон и дата (по ним позже находится подписанный бланк в архиве); фото и № бланка не хранятся.
- **US-10.** Видеть свои поданные лиды и статус отправки. — *Критерии:* список лидов менеджера; индикатор «отправлено / ошибка»; повтор при ошибке.

### Администратор (аналитика + настройка)
- **US-11.** Дашборд лидов по каналам, менеджерам и выставкам. — *Критерии:* счётчики и разрезы по форме/менеджеру/выставке; выбор периода; данные сходятся с Bitrix.
- **US-13.** Выгрузить данные. — *Критерии:* экспорт выборки (CSV/Excel) с полями и пометкой формы.
- **US-14.** Управлять аккаунтами менеджеров. — *Критерии:* создание/деактивация аккаунта; назначение и отзыв личной ссылки; маппинг на ID пользователя Bitrix; неактивный аккаунт/отозванная ссылка не дают входа.
- **US-15.** Настраивать списки значений и проверять связь с полями Bitrix. — *Критерии:* списки редактируются в одном месте; значения отражаются в форме; маппинг на коды Bitrix виден.

*(US-12 удалена — рекламные согласия.)*

### Сквозные (Система)
- **US-16.** Отправлять лид в Bitrix с корректным форматом мультиполей PHONE/EMAIL. — *Критерии:* телефон в карточке кликабельный; email распознаётся как контакт.
- **US-17.** Записывать факт согласия ПДн и дату/время. — *Критерии:* в лиде видно состояние согласия и дату (дата + телефон = ключ для поиска бланка).
- **US-18.** Дедуплицировать по телефону/email между каналами. — *Критерии:* повтор того же контакта не создаёт второй лид (склейка/предупреждение).

*(US-19 удалена — защиту от спама не делаем.)*

## 6. Критерии приёмки / DoD (эпик целиком)

- Лид из каждого из трёх каналов доходит до Bitrix с маппингом полей и корректным форматом телефона/email.
- Обязательны только Имя + хотя бы один контакт (телефон или email) + согласие ПДн; остальные поля опциональны.
- Согласие ПДн блокирует отправку без него; записано с датой. По телефону + дате согласие находится и в системе, и в бумажном архиве.
- Лид канала «бумага» закреплён за залогинившимся менеджером (`ASSIGNED_BY_ID`); фото и № бланка не хранятся.
- Каждый лид помечен источником «Выставка» и (желательно) формой — существующими полями Bitrix.
- Дедупликация работает между каналами.
- Распознавание/хранение — на РФ-инфраструктуре; фото бланка не сохраняется.
- Вход менеджера — по личной ссылке; админ может назначить и отозвать её.
- Дашборд показывает корректные разрезы по форме/менеджеру/выставке.
- Подпись Telegram проверяется.
- Списки значений совпадают в бланке, форме и Bitrix.

## 7. Зависимости / риски

**Зависимости:**
- Одобрение размещения формы на сайте iZBURG (иначе — сайт дашбордов; проверить, что политика доступна и там).
- Текст согласия/политики — уже на сайте iZBURG; использовать ссылку.
- Утверждение набора полей отделом продаж.
- Подтвердить существующие поля Bitrix для источника «Выставка» и формы (веб/тг/бумага).
- Бумажный архив должен быть организован так, чтобы бланк находился по телефону + дате (офлайн-процесс бухгалтерии).
- Выбор РФ-провайдера хостинга и OCR.
- Новые поля Bitrix НЕ требуются.

**Риски:**
- Качество распознавания рукописных контактов → проверка менеджером + клеточки под телефон.
- Личная ссылка как единственный фактор входа → митигируется: не пересылать, возможность отзыва/перевыпуска админом. Для текущего масштаба приемлемо.
- Трансграничная передача ПДн при зарубежном OCR → закрыто требованием РФ-инфраструктуры.
- Ручной ретеншен/удаление → зависит от дисциплины; для текущего масштаба принято осознанно.

## 8. Оценка / приоритет / этап

- **Приоритет 1** (к ближайшей выставке): веб-форма на сайте iZBURG + согласие ПДн + минимальная обязательность полей + отправка в Bitrix + шаблон бумажного бланка (US-1, 3, 4, 16, 17).
- **Приоритет 2:** режим менеджера (вход по личной ссылке) + распознавание без хранения фото + привязка (US-5…10), дедупликация (US-18).
- **Приоритет 3:** аналитика и настройка (US-11, 13, 14, 15), Telegram-канал (US-2).
- Размер: средний; основная сложность — режим менеджера с OCR. Telegram-канал опционален.
- Оценка в часах — не определено (зависит от выбора провайдеров и готовности списков значений).

## 9. Метрики успеха

- Доля посетителей, чьи контакты дошли до Bitrix — рост.
- Доля лидов с корректным дозвонным телефоном.
- Время от выставки до появления лида в Bitrix (для бумаги — в тот же вечер).
- Доля лидов с зафиксированным согласием ПДн = 100%.
- Распределение лидов по формам и менеджерам.
